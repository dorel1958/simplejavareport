Help

To create a new report:

1. File/New will generate an empty report with 3 bands: pageHeader, detailBand and pageFooter.
You should name and save the XMLfile now.

2. Report/Page Setup allow you select paper size, margins and orientation.

3. Report/Default font and color allow you select the initial font for text controls, and color for text and line controls.

4. Report/Bands allow you select the bands you want in your report (to resize a band drag the band separator).

5. Report/Test Data put some test data that will allow you to preview the report in design mode.
There are 3 types of data:
- mCommon - a Map<String, String> with pairs key-value. The keys will be the same at runtime, but the values will change.
These values can be used in any band.
- columnNames - a String[] with the column names, used only in design mode to identify columns by name.
- lData - a List<String[]> with 'lines' (String[]) that contain table data that will be changed at runtime.
Values can be used ONLY in detail Band or summary Band.
of course 'columnNames' has the same length and order as every 'line' in lData.
ATTENTION - if you modify MapKeys or columnNames later you will have to modify them to all controls that uses them!

6. RightClick inside a band to add controls

7. Click to select control, ShiftClick to multiselect, Drag, starting from free space, to select controls.
Drag, or use arrow keys, to move controls.
Drag handlers, or Shift + arrow keys, to resize control.
Use menu Format to arrange controls.

8. DubleClick control to edit properties:


for TextControl

Text source
Text - the exact text to print.
Common - from Map mCommon, the Key of the Value to print.
Data - from List<String[]> lData, the columnName to print in detail band.
Report - the name of the ReportVariable to print.
Numeric value - if the string to print represent a number (if number length > control width, will replace printed value with '...').
text fields in Summary band will print the SUM of the column values in lData (converting String to Number).
...

for ImageControl

Text source
Text - the exact image source fileName.
Common - from Map mCommon, the Key of the Value that contain image source fileName (not in design mode).
Data - from List<String[]> lData, the columnName that contain image source fileName (not in design mode).
If source and frame are different sizes:
CLIP_CONTENTS - print only the part of the image that fit into frame.
SCALE_FILL_THE_FRAME - scale image to fill the frame.
SCALE_RETAIN_SHAPE - scale image to fit entirely in the frame, no deformations.

for LineControl

Line type can be:
line (with end type:'CAP_SQUARE', 'CAP_ROUND', 'CAP_BUTT'), flip - reverse angle.
rectangle (with join type:'JOIN_MITTER', 'JOIN_ROUND', 'JOIN_BEVEL')
...

9. Report/Preview to see the report containing test data

preview command save the report to a xml file.

10. at runtime
add SimpleJavaReport.jar to your projectLibraries;
create a Map with the same keys as mCommon but with desired values;
create a List<String[]> and populate it with desired values (in columnNames order from XML reportFile);
create an instance of JavaReport (all parameters can have null values) and preview or print the report;
if mCommon or lData are not null, they will replace design-time test values.
if numeFisXML is null or wrong you will preview or print an empty page.

Example:

import dorel.simplejavareport.report.JavaReport;

Map<String, String> mCommon = new HashMap<>();
mCommon.put("DesignTimeKey", "RuntimeValue");
...
List<String[]> lData = new ArrayList<>();
String[] line1 = {"Col1 runtimeValue", "Col2 runtimeValue", "Col2 runtimeValue", "Col3 runtimeValue", ...};
lData.add(line1);
...
JavaReport jr=new JavaReport(parentFrame, numeFisXML, mCommon, lData);
jr.preview();
// or
jr.print();
...
